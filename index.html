Introdução

Bem vindo a documentação da API do conciliador Audit All!


Você pode usar nossa API para consumir todos os registros dos seus recebimentos e das suas vendas totalmente tratados e padronizados, facilitando a integração entre seu sistema e o conciliador. As infomrações contidas nas coleções, são as mesmas informadas nos relatórios da aplicação.

Para o consumo das informações , você precisará utilizar os dados de login da plataforma (CNPJ, Usuário e Senha) para gerar o Token e assim enviar nas requisições juntamente com os filtros. Sempre estará disponível no sistema as transações realizadas em D-1 e os registros financeiros estarão disponíveis e D.

Não fazemos nenhum tipo de restrição referente a limites de tempo para consumo. As infromações são processadas e disponibilizadas 1 vez por dia e assim não se faz necessário o consumo em vários momentos do mesmo dia.

Reforçamos a base das informações são dos extratos enviados pelas adquirentes e que caso haja alguma divergência, o responsável pelo estabelecimento terá que entrar em contato direto com a operadora para solicitar o reembolso.

O padrão utilizado pela API é o REST e o formato JSON codificado em UTF-8 para receber e retornar os dados e processamento das requisições do tipo POST.

Neste documento você encontrara todas as informações técnicas necessárias para integração com a aplicação.

Autenticação

Sempre que precisar consumir os dados da API, você precisará enviar uma solicitação de autenticação para receber Token de acesso.
Os dados de envio são os mesmos de acesso a plataforma Web (CNPJ, Login e Senha). Com a chave de retorno você inclui nas chamadas do sistema. A API retornará todos os dados cadastrais da sua conta.

Nunca compartilhe suas credenciais! Mantenha seus dados sempre protegidos e seguros.
